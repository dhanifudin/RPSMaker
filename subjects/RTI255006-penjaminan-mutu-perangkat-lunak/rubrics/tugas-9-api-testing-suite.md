# Tugas 9: API Testing Suite

**Bentuk Tugas/Evaluasi:** Hands-on Practice

**Sub-CPMK:** SCPMK608-04203

## Deskripsi
Mahasiswa membuat API testing suite manual dan otomatis.

## Metode Pengerjaan
Membuat request, environment, assertion, negative case, dan automated run.

## Bentuk Format Luaran
Postman/Newman collection dan laporan eksekusi.

## Rubrik Penilaian

| Indikator Penilaian | Bobot (%) | Kriteria 3 (Baik) | Kriteria 2 (Cukup) | Kriteria 1 (Kurang) |
| --- | ---: | --- | --- | --- |
| Desain request dan environment | 1.2 | Collection, endpoint, method, parameter, body, header, token, dan environment disusun benar. | Request berjalan, tetapi environment, token, atau parameter belum konsisten. | Request tidak sesuai kontrak API atau sulit dijalankan ulang. |
| Assertion dan negative case | 1.1 | Assertion status, schema, payload, business rule, serta negative case mencakup kontrak API. | Assertion tersedia, tetapi negative case atau validasi schema belum lengkap. | Assertion tidak memadai atau hanya mengecek response dasar. |
| Automation dan laporan eksekusi | 0.7 | Newman/automation berjalan, laporan hasil tersedia, dan failure dianalisis. | Automation berjalan sebagian, tetapi laporan atau analisis failure belum lengkap. | Tidak ada automation/evidence eksekusi yang valid. |
