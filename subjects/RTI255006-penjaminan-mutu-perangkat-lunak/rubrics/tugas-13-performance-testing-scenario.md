# Tugas 13: Performance Testing Scenario

**Bentuk Tugas/Evaluasi:** Hands-on Practice

**Sub-CPMK:** SCPMK608-04204

## Deskripsi
Mahasiswa membuat dan menjalankan skenario performance testing.

## Metode Pengerjaan
Mendesain skenario beban, menjalankan tools, membaca metrik, dan membuat rekomendasi.

## Bentuk Format Luaran
Skenario performance testing dan laporan metrik.

## Rubrik Penilaian

| Indikator Penilaian | Bobot (%) | Kriteria 3 (Baik) | Kriteria 2 (Cukup) | Kriteria 1 (Kurang) |
| --- | ---: | --- | --- | --- |
| Desain skenario beban | 1.2 | Jumlah user, ramp-up, workload, data, SLA, dan target metrik dirancang realistis. | Skenario beban tersedia, tetapi SLA, data, atau workload belum lengkap. | Skenario tidak realistis atau tidak dapat dijalankan. |
| Eksekusi tools dan pengumpulan metrik | 1.1 | JMeter/k6 dijalankan benar, metrik response time, throughput, error rate, dan resource dicatat. | Tools berjalan, tetapi beberapa metrik atau konfigurasi belum lengkap. | Tools tidak berjalan atau metrik tidak valid. |
| Analisis bottleneck dan rekomendasi | 0.7 | Bottleneck dianalisis dari metrik dan rekomendasi teknis disusun jelas. | Analisis tersedia, tetapi penyebab bottleneck atau rekomendasi masih umum. | Hasil performa tidak dianalisis. |
