# Pertemuan 10 - Tahapan implementasi basis data

## Kemampuan Akhir
- SCPMK1001-01501: Mampu mengimplementasikan skema basis data dan menyusun query SQL untuk mendefinisikan, memanipulasi, dan menampilkan data sesuai dengan studi kasus.

## Materi Pembelajaran
Tahapan implementasi basis data: membuat basis data, tabel, atribut, primary key, foreign key; bahasa SQL dan SQL-DDL (CREATE, ALTER, DROP).

## Bentuk dan Metode Pembelajaran
Bentuk: ceramah, diskusi, latihan analisis. Metode: Case Based Learning (CBL). Penugasan: Tugas 8 menuliskan perintah DDL untuk mengimplementasikan hasil rancangan basis data.

## Estimasi Waktu
1 x 2 x 50' tatap muka dan tugas terstruktur; 1 x 2 x 50' tugas mandiri.

## Pengalaman Belajar Mahasiswa
Mahasiswa menjelaskan tahapan implementasi basis data serta menuliskan perintah SQL-DDL untuk mengimplementasikan dan mengelola basis data, tabel, atribut, dan relasi.

## Penilaian
Kriteria: ketepatan dan penguasaan. Bentuk: keaktifan diskusi kelompok; ketepatan jawaban tugas.

## Indikator Penilaian
Ketepatan, kelengkapan, dan kerapian perintah SQL-DDL untuk implementasi rancangan; ketepatan perintah mengubah dan menghapus basis data, tabel, dan relasi.
