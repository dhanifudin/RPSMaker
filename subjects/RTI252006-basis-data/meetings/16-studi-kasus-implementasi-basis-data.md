# Pertemuan 16 - Studi kasus implementasi basis data

## Kemampuan Akhir
- SCPMK1001-01502: Mampu menganalisis kebutuhan data, merancang skema basis data, dan menyusun query SQL secara efisien untuk mengelola basis data sesuai dengan studi kasus.

## Materi Pembelajaran
Studi kasus implementasi basis data: pemetaan ERD ke model relasional (3NF), implementasi ke MySQL dengan DDL, manipulasi data dengan DML, dan query SELECT termasuk multi table.

## Bentuk dan Metode Pembelajaran
Bentuk: ceramah, diskusi, latihan analisis. Metode: Case Based Learning (CBL). Penugasan: Tugas laporan perancangan dan implementasi basis data.

## Estimasi Waktu
1 x 2 x 50' tatap muka dan tugas terstruktur; 1 x 2 x 50' tugas mandiri.

## Pengalaman Belajar Mahasiswa
Mahasiswa memetakan rancangan ke model relasional, mengimplementasikan ke DBMS MySQL, memanipulasi data, dan menampilkan data dari beberapa tabel.

## Penilaian
Kriteria: ketepatan dan penguasaan. Bentuk: keaktifan diskusi kelompok; ketepatan laporan studi kasus.

## Indikator Penilaian
Ketepatan query DDL, DML, dan SELECT untuk implementasi dan penampilan data hasil rancangan.
